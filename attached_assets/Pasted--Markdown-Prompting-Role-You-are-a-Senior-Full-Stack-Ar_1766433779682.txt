# **Objective:**
Design and build a **fully functional, production-level application** that allows users to upload **credit card statements or bank financial statements**, extract standardized transaction data, and **download the results securely** as **Excel (.xlsx) or Word (.docx)** files.  
The application must work on **Web and Mobile (Android & iOS)** and be architected for **future App Store and Play Store deployment**.

---

# **Context:**
The application processes **highly sensitive financial data**, similar to systems used by Tier-1 banks.  
It must:
- Handle multiple statement formats
- Ensure strict data isolation per user
- Prevent data breaches
- Follow best practices for secure file handling, encryption, and access control  
This is **not an MVP**—it is a **production-ready, enterprise-grade system** built to scale and withstand audits.

---

# **Instructions:**

### **Instruction 1: Core Functionality & Data Handling**
- Allow users to securely upload **bank or credit card statements** (PDF format).
- Parse, extract, and normalize financial transaction data.
- Extract **only the following standardized fields**:
  - **Date of Transaction**
  - **Transaction Details**
  - **Transaction Amount (USD)**
- Normalize date formats, currency values, and text descriptions.
- Maintain transaction order as shown in the original statement.

---

### **Instruction 2: Output Generation & Download**
- Allow users to select an output format within the app:
  - **Excel (.xlsx)**
  - **Word (.docx)**
- Generate the selected document dynamically.
- Provide a **secure, time-limited download option**.
- Ensure:
  - Files are isolated per user
  - No cross-user data access
  - Automatic file cleanup after download or expiration

---

### **Instruction 3: Architecture, Security & Platform Design**
- Design a **modular, scalable architecture** supporting:
  - Web application (responsive)
  - Mobile applications (Android & iOS)
  - Shared backend services
- Enforce **bank-grade security**:
  - Encryption in transit (TLS)
  - Encryption at rest (AES-256)
  - Secure authentication & role-based access control
  - Secure file storage and lifecycle management
  - Audit logs and monitoring
- Ensure cloud-native design (AWS / GCP / Azure ready).
- Architect for future enhancements such as OCR, AI-based parsing, multi-currency, and multi-language support.
